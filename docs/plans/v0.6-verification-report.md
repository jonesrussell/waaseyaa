## Risk Register

1. CI environment drift risk (Medium)
- Playwright/browser and PHP toolchain updates can shift runtime behavior.
- Mitigation: lock versions in CI setup and keep gate suites deterministic.

2. Fixture expansion risk (Medium)
- Adding new deterministic fixtures can invalidate assumptions in brittle tests.
- Mitigation: keep fixture pack canonical and update tests to assert invariants rather than hardcoded row counts where appropriate.

3. Architectural boundary compliance regression (Low)
- Rule: reusable primitives must remain in Waaseyaa packages; Minoo should hold only domain config/templates/fixtures/wiring.
- Current state: extraction passes completed; reusable relationship/workflow/validation/delete-guard primitives now live in framework packages (`waaseyaa/relationship`, `waaseyaa/workflows`).
- Mitigation: enforce review checklist on future PRs to prevent logic drift back into app layer.
