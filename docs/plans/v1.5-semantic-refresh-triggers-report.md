# v1.5 Semantic Refresh Triggers Report (#168)

## Goal
Implement deterministic semantic refresh trigger rules tied to ingestion events with stable categories, precedence, and diagnostics.

## Delivered
- Added `SemanticRefreshTriggerPlanner`:
  - canonical category precedence:
    1. `provenance_change`
    2. `relationship_change`
    3. `policy_change`
    4. `structural_drift`
  - deterministic trigger diagnostics under `refresh.*`.
- Wired planner into `ingest:run`:
  - new options:
    - `--refresh-baseline`
    - `--refresh-snapshot-output`
  - emits:
    - `diagnostics.refresh`
    - `diagnostics.refresh_summary`
    - `meta.refresh_required`
    - `meta.refresh_primary_category`
- Added snapshot builder + baseline loader for repeatable diff semantics.
- Added spec:
  - `docs/specs/semantic-refresh-trigger-contract.md`
