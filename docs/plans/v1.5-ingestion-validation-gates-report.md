# v1.5 Ingestion Validation Gates Report (#165)

## Goal
Enforce deterministic post-ingest validation gates across workflow, visibility, and semantic invariants before publishable mapped output is emitted.

## Delivered
- Added `ValidationGateValidator`:
  - workflow rules (`validation.workflow.*`)
  - visibility rules (`validation.visibility.*`)
  - semantic rules (`validation.semantic.*`)
- Added `ValidationDiagnosticEmitter` with fixed-template, category-aware diagnostics and deterministic ordering.
- Integrated validation gates into `ingest:run` lifecycle:
  - schema pass -> map pass -> validation gate pass
  - `atomic_fail_fast` blocks mapped output on validation errors
  - `validate_only` runs gates but emits no mapped output
- Added `diagnostics.validation_summary` with deterministic categories/codes.
- Added spec:
  - `docs/specs/ingestion-validation-gates-contract.md`
